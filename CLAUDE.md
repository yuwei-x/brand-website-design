# CLAUDE.md — 前端网站设计规则

## 第一步

- **写前端代码前**，先调用 `frontend-design` skill —— 每个会话都要，无例外。
- **视频转网站时**，调用 `yuwei-video-to-website` skill —— 它会处理抽帧、Canvas 渲染、滚动动效全流程。
- 当 Skill 中的专业规范与本文件的通用规则冲突时，以 Skill 为准。

## 参考图片

- 如果提供了参考图片：精确匹配布局、间距、排版和配色。内容用占位符（图片用 `https://placehold.co/WIDTHxHEIGHT`，文案用临时文字）。不要擅自增加或「改进」设计。
- 如果没有参考图片：从零开始设计，遵循下方 Anti-Generic 规则。
- 截图你的输出，与参考图片对比，修正差异，再截图。至少做 2 轮对比。直到没有可见差异、或用户明确说可以了才停。

## 本地服务器

- **必须通过 localhost 访问** —— 永远不要截图 `file:///` URL。
- 启动方式：`python3 -m http.server 3000` 或 `npx serve .`（默认端口 3000）。
- 如果服务器已经在运行，不要启动第二个实例。

## 截图工作流

- 使用浏览器手动截图或 Agent 自带截图能力。
- 截图后仔细对比：间距/内边距、字号/字重/行高、颜色（精确到 hex）、对齐、圆角、阴影、图片尺寸。
- 对比时要具体：「标题是 32px 但参考图约 24px」「卡片间距 16px 但应为 24px」。

## 输出默认值

- 单个 `index.html` + `css/style.css` + `js/app.js`，除非用户另有要求
- 滚动驱动网站使用纯 CSS + GSAP（不用 Tailwind），其他类型网站可用 Tailwind CSS via CDN
- 占位图片：`https://placehold.co/WIDTHxHEIGHT`
- 移动优先响应式

## 品牌素材

- 开始设计前先检查 `brand_assets/` 文件夹。里面可能有 logo、配色指南、风格指南或图片。
- 如果有素材就用，不要在有真实素材的地方用占位符。
- 如果有 logo 就用它。如果定义了配色方案，用那些精确值，不要自己编造品牌色。

## Anti-Generic 规则（避免千篇一律的 AI 风格）

以下规则是通用底线。Skill 会根据具体风格方向细化或覆盖这些规则。

### 颜色
不要用默认色板偷懒。选一个品牌色，从它派生出整个色系。

### 阴影
用多层阴影，带颜色倾向，低透明度。不用扁平的 `shadow-md`。

### 排版
标题和正文的视觉层次必须清晰 —— 通常通过不同字体实现，但 Apple 极简风格允许同族字体配合不同字重。
- 大标题：紧凑字间距（`letter-spacing: -0.03em`）
- 正文：宽松行高（`line-height: 1.7`）

### 动效
只动 `transform` 和 `opacity`。**永远不要 `transition-all`**。

### 交互状态
每个可点击元素**必须有** hover、focus-visible 状态。

### 间距
使用有意识的、一致的间距系统 —— 不要随机选值。

## 绝对禁止

- 不要添加参考图中没有的区块、功能或内容
- 不要「改进」参考设计 —— 精确匹配
- 不要只截图一次就停
- 不要用 `transition-all`

## 沟通风格

- **语言**：所有输出使用中文
